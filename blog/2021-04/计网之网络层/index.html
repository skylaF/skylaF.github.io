计网之网络层
  • Apr 23, 2021
网络层向上只提供简单灵活的、无连接的、尽最大努力交互的数据报服务。
与IP协议配套使用的还有三个协议：  
  - 地址解析协议ARP  
  - 网际控制报文协议ICMP（ping程序就是使用这个协议）  
  - 网际组管理协议IGMP

​
IP协议

1. IP网际协议


IP首部格式

1.1. IP数据报首部格式

  • 首部长度：20-60字节，固定长度20字节，其他为可选字段
  • 版本： 4位
  • 首部长度：4位，单位位32位，最大可以到15个32位字长；
  • 区分服务：8位，
  • 总长度： 16位，总长度 = 首部 + 数据长度
  • 标识： 16位，计数数据报分片个数
  • 标志： 3位，表示是否还有分片MF，以及是否分片DF（don’t Fragment）
  • 片偏移： 13位，某片在原分组中的相对位置。
  • 生存时间：8位，TTL，现在是以跳数作为计算单位的。
  • 协议： 8位，指明该数据报应该上交给哪个协议处理，有ICMP、IGMP、TCP、UDP
  • 首部检验和：16位，只检验首部而不包含数据部分，一般不用CRC，二是用位与运算和位取反运算进行
  • 源地址：32
  • 目的地址：32

1.2. IP编址方式

分类->子网划分->无分类

1.2.1. 分类

  • 由网络号和主机号组成，共32位
  • 分成五类
    • A类：0-127
    • B类：128-191
    • C类：192-223
    • D类：224-239
    • E类：240-255

1.2.2. 子网划分

  • 原因：
    • IP地址空间利用率低
    • 两级IP地址不够灵活：如果是子网，划分的工作就由单位内部自己做。
  • 思路：从主机号取用若干位作为子网号，形成{<网络号>,<子网号>,<主机号>}
  • 子网掩码：使用子网掩码可以确定主机号

1.2.3. 无分类编址CIDR（构造超网/sider/）

  • 从三级编址变回无分类两级编址，即{<网络前缀>,<主机号>}
  • 使用斜线记法，例如128.14.35.2/20，在表示前20位相同的可以组成一个CIDR地址块
  • 虽然不使用子网了，但是仍然由地址掩码，斜线后面的数字就是地址掩码的位数
  • 一个CIDR块中有很多地址，形成了路由聚合，即超网
  • 最长前缀匹配
    • 路由器在转发时，可能得到多个匹配结果，那么它只选择具有最长前缀的匹配结果，这就是最长匹配。

2. 三个配套协议

ICMP、ARP、IGMP


各层使用协议

2.1. ARP地址解析协议

  • IP数据报往下需要包裹着MAC硬件地址，形成MAC帧，交给数据链路层，在链路层传送时，IP数据报不需要改动，但是MAC帧的地址会发生变化。
  • ARP协议就是用来找IP地址对应的MAC地址，它还是在网络层使用的
  • 每个主机都有一个ARP高速缓存cache，里面存放着IP地址与MAC地址的映射表
  • 查找IP与对应MAC地址时有两种情况
    • 直接在映射表中找到；
    • 映射表中没有信息，则本机运行ARP进程，向本局域网广播一个请求分组，对应主机接收到之后单播一个响应分组，发起主机接收后在cache种写入映射。

2.2. ICMP网际控制报文协议

  • 更有效地转发IP数据报和提高交付成功的机会
  • ICMP封装在IP数据报的数据部分，但是不属于高层协议
  • 分为ICMP差错报告报文和ICMP询问报文
      1. ICMP差错报告报文：
      • 终点不可达
      • 时间超过
      • 参数问题
      • 改变路由
      1. ICMP询问报文：
      • 回送请求或回答Echo
      • 时间戳请求或回答
  • ICMP的应用
    • PING：使用ICMP的回送请求和回送回答报文，PING程序直接使用ICMP，而不通过运输层的TCP或UDP
    • trackroute：

2.3. IGMP网际组管理协议

使用IP数据报传递报文，用于IP多播，主要是让连接在本地局域网上的多播路由器知道著否有主机参加某个多播组
  • 要加入多播组的成员，向多播路由器发送IGMP报文，路由器收到后利用路由选择协议把成员关系转发给其他多播路由器
  • 周期性的探询本局域网上的主机，有回应的主机被认为某个组是活跃的，如果没有任何回应就认为这个组已经离开了，然后转发这种关系给其他多播路由器

3. 路由选择协议

内部网关协议IGP：RIP、OSPF
外部网关协议EGP：BGP

3.1. 内部网关协议-RIP（封装在UDP数据报中）

  • 基于跳数的距离向量协议，经过一个路由器就是跳数加一，最多15，16表示不可达。
  • 特点：
    • 仅和相邻的路由器交换信息，交换的是自己的路由表
    • 按固定时间间隔交换信息
    • 路由表如下：
      目的网络 距离 下一跳路由器
      Net1 3 R1
      Net2 4 R3
    • 收到RIP报文之后，把没有的项目加到路由表种中，而存在的项目只更新比原有距离更短的，更新后的路由器都是最短距离。
    • 缺点：跳数只有15限制了网络的规模，而且网络不流畅时需要较多的时间才能将消息更新到所有路由器。

3.2. 内部网关协议OSPF（直接使用IP数据报）

  • 开放最短路径优先，使用Dijkstra算法。
  • 使用洪泛法，向所有路由器发送信息；所有相邻路由器的链路状态；只有链路发生变化时，才使用洪泛法发送消息
  • 特点：
    • 有负载平衡的能力。

3.3. 外部网关协议BGP（使用TCP连接）

  • 选择可达且比较好的路由，而不是寻找最佳的路由，采用路径向量路由选择协议
  • BGP发言人：就是代表这个系统与相邻系统交流的路由器，可以是边缘路由器也可以不是
  • 四种报文：
    • OPEN报文：与另一个BGP发言人建立联系
    • UPDATE报文：通告某一路由的信息
    • KEEPALIVE报文：周期性证实邻站的连通性
    • NOTIFICATION报文：通知检测到的差错

4. 路由结构和流程

路由器任务是转发分组，结构分成两大部分：路由选择和分组转发。

4.1. 路由选择

  • 由路由选择协议构造出路由表，具体看上面第3节

4.2. 分组转发

由三部分组成：交换结构、一组输入端口、一组输出端口
  • 交换结构：根据转发表对分组进行处理/转发和路由选择是不同的，转发表仅涉及一个路由器，但是路由选择涉及很多路由器。
    • 通过存储器；
    • 通过总线；
    • 通过互联网络
  • 输入输出：分别有物理层、链路层、网络层的处理模块，物理层进行比特的接收。链路层按协议接收传送分组的帧，去掉帧的首部和尾部后，交给网络层处理模块。

5. IPv6

数据单元PDU称为分组，而不是数据报；32位变为128位；数据报分为基本首部和有效载荷。
  • 首部长度固定为40字节
  • 增加了分片扩展首部，不需要标识、标志和片偏移
  • 六种扩展首部：
    • 逐跳选项；
    • 路由选择；
    • 分片；
    • 鉴别；
    • 封装安全有效载荷；
    • 目的站选项
  • 三种基本类型地址：
    • 单薄：点对点通信
    • 多播；发送到一组计算机中的每一个
    • 任播；终点是一组计算机，但是数据报只交付给其中一个，一般是距离最近的
  • 冒号十六进制记法：每16为16进制值表示，间隔不再是“.”而是“:”，而且允许省略0，叫做零压缩
  • 从IPv4过渡到IPv6
    • 双协议栈，同一台主机用IPv4/IPv6
    • 隧道技术：将IPv6数据报要封装在IPv4中，成为IPv4的数据部分，只需要在协议字段部分表明协议是IPv6就可以让路由器知道转成IPv6

6. 其他

  • 网络地址转换NAT
    专用网内部的主机使用本地 IP 地址又想和互联网上的主机通信时，可以使用 NAT 来将本地 IP 转换为全球 IP。 主要是在路由器上安装NAT软件，一个NAT路由器可以有多个IP地址，所以拥有 n 个全球 IP 地址的专用网内最多只可以同时有 n 台主机接入互联网。 通过NAT路由器的通信必须由专用网的主机发起，因为NAT路由器收到外网发起的通信也不知道要发给哪个主机。
    • 现在常用的 NAT 转换表把传输层的端口号也用上了，使得多个专用网内部的主机共用一个全球 IP 地址。使用端口号的 NAT 也叫做网络地址与端口转换 NAPT
      专用网内的主机发起时带上端口号，然后NAPT将其转换成一个公网IP以及一个端口，所以收到外网的数据报时根据不同的端口号就能找到正确的目的主机。
  • 多协议标记交换MPLS
    在MPLS的上层可以使用多种协议，MPLS利用面向连接技术，使每个分组携带一个叫做标记的整数，交换机读取分组的标记，用标记值来加锁分组转发表，比路由表来转发分组要快得多
