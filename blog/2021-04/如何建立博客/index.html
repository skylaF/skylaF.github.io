使用hugo建立博客

正常操作

  • 下载hugo（这个官网有相关教程）
  • 使用以下命令建立本地site目录

hugo new site siteName

  • 下载主题themes,建议使用以下三个主题

git clone https://github.com/orianna-zzo/AllinOne.git themes/AllinOne
git clone https://github.com/CaiJimmy/hugo-theme-stack/ themes/hugo-theme-stack
git clone https://github.com/olOwOlo/hugo-theme-even themes/even

  • 直接复制主题目录下的config.toml或config.yaml文件到siteName目录下，根据主题的描述自行更改配置变量
  • 使用以下命令建立你文章文档（md格式文件），生成的文件就在conten目录下，一般可以在content目录下建议对应的文件夹

hugo new newContent.md

  • 可以使用以下命令热部署你的博客内容

hugo server –watch

托管到github或gitee上（也就是让别人可以看到你的博客而不需要自己买服务器）

  • 先学会使用git还有github
  • 在github上，建立一个新的仓库，名字取做 yourGithubName.github.io
  • 在本地的博客目录下使用git与这个仓库建立联系，使用以下一系列命令

git init
git config –global user.name"yourGithubName"
git config –global user.email"youremail@eamil.com"
git remote add origin git@github.com:yourGithubName/yourBlogName.git(复制仓库的链接)
git remote -v
git pull origin master
git add .
git commit -m"describe your commit"
git push origin master
git log

  • 这样只要没报错就能使用yourGithubName.github.io查看你的博客了

可能遇到的问题

  1. 写出来的md文件发布之后没有格式
  • 如果你使用的不是Typora软件来写md文件，那么建议使用Typora软件，基本没有问题。
  • 如果你使用的是Typora软件，那么可以试试在每一行之后加两个空格。可以保证在发布之后是有格式的。
  1. 写出来的文章中文乱码
  • 可以将文件的编码修改成UTF-8，一般就可以解决问题了，相关回答
