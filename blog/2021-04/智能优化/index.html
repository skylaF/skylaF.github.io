算法内容

  • 遗传算法
    1. 理论基础：模式定理——具有相似结构特点的个体编码字符串，阶越高，样本就越少，确定性也越高。
    2. 内容：使用群体搜索技术，将种群代表一组问题解，通过对当前种群施加选择、交叉和变异等操作来产生新一代种群，并逐步使种群进化到包含近似最优解的状态。
    3. 操作：
      1. 选择：运用基于轮盘赌的方法，也就是比例选择，个体适应度越大，被选中的概率就越大，所谓适应度就是在种群中的数量，或者用适应度函数来计算。
      2. 交叉：对每一个个体，根据交叉概率交换它们之间的部分染色体。
      3. 变异：根据变异概率将某一个或某一些基因座上的基因值改变为其他的等位基因值，有二进制变异（值取反）和实值变异（取范围内的其他随机值）。
    4. 特点：（并行、高效、全局搜索方法）
      1. 以决策变量的编码作为运算对象，对一些只有代码概念而无数值概念的优化问题，显示出优越性。
      2. 直接以目标函数值作为搜索信息，而不需要目标函数的导数值等辅助信息，对于不能求导或很难求导的函数问题就具有优越性。
      3. 同时使用多个搜索点的搜索信息，是从一个由很多个体组成的初始种群开始的，而不是从单一的个体开始的，这里面包括很多群体信息，避开搜索一些不必要的点，这就是隐含的并行性。
      4. 算法是基于概率进行的，也就是自适应概率搜索技术，从而增加了算法的灵活性。与其他算法 相比，遗传算法的鲁棒性是的参数对其搜索效果的影响尽可能小。
      5. 遗传算法具有自组织、自适应和自学习等特性。同时遗传算法具有可扩展性，易于同别的算法结合，生成综合双方优势的混合算法。
  • 差分进化算法
    1. 内容：通过把种群中的两个成员之间的加权差向量加到第三个成员上来产生新的参数向量，即“变异”，然后将变异向量的参数与另外预先确定的目标向量参数按一定规则混合来产生实验向量，即“交叉”；若试验向量的代价函数比目标向量的代价函数低，试验向量成为下一代的目标向量，，即“选择”
    2. 操作：
      1. 初始化
      2. 变异
      3. 交叉
      4. 选择
      5. 边界条件处理：必须保证产生新个体的参数值位于问题的可行域中。
    3. 特点：
      1. 结构简单，容易使用，差分变异算子只涉及向量的加减运算，所以容易实现；控制参数少
      2. 性能优越。差分进化算法具有良好的可靠性、高效性和鲁棒性，对于大空间、非线性和不可求导的连续问题。其求解效率比其他进化方法好
      3. 自适应性。差分进化算子可以是固定常数，也可以具有变异步长和搜索方向自适应的能力，根据不同目标函数自动调整从而提高搜索质量。
      4. 具有内在的并行性，可协同搜索，在同等精度要求下，差分进化算法具有更快的收敛速度。
      5. 算法通用。可直接对结构对象进行操作，不依赖于问题信息，不存在对目标函数的限定。

  • 免疫算法

    1. 内容：采用群体搜索策略，通过迭代计算，最终以较大的概率得到问题的最优解。
      1. 抗原识别和初始抗体产生（种群）
      2. 抗体评价。
      3. 免疫操作：免疫选择、克隆、变异、克隆抑制、种群刷新
    2. 具体：
      1. 主要在免疫操作:
        • 免疫选择：根据种群中抗体的亲和度和浓度计算结果选择优值抗体，使其活化（亲和度根据亲和度函数计算，函数则是在抗原识别的时候构造）；
        • 克隆：对活化的抗体克隆若干副本；
        • 变异：对克隆之后的副本进行变异操作，使其发生亲和度突变；
        • 种群刷新：对变异结果进行再选择，抑制亲和度低的抗体，保留亲和度高的变异结果。
    3. 特点：
      1. 全局搜索能力：
      2. 多样性保持机制：对抗体进行浓度计算（浓度表征抗体的多样性，浓度越高多样性就越差，导致搜索会集中于一个区域，不利于全局优化），浓度高的抗体被抑制，保证算法全局收敛性能的一个重要方面。
      3. 鲁棒性强：不针对特定问题，而且不强调算法参数设置和初始解的质量，具有很强的适应性和鲁棒性。
      4. 并行搜索机制：免疫算法不需要集中控制，可以实现并行处理，而且是一种多进程的并行优化。适合于多模态的优化问题。

  • 蚁群算法（信息素是在所有蚂蚁都周游一次之后才更新的）

    1. 内容：模拟蚂蚁集体寻径行为而提出的一种基于种群的启发式随机搜索算法，蚂蚁可以通过信息素进行信息传递，某一路径上经过的蚂蚁越多，后来者选择该路径的概率就越大。
    2. 蚂蚁的一步转移概率由两个参数决定：①信息素值；②先验值。
    3. 信息素的更新方式：①挥发，信息素以一定的比率减少；②增强，走过的边增加信息素值（所有蚂蚁走过一遍才更新，而且基本算法中最佳路线的信息素没有特别的处理）。
    4. 改进算法（对需要较长时间和容易出现停滞现象的改进）：
      1. 精英蚂蚁系统：将最好解称为best-so-far，修改这条路径的信息素时，会释放额外的信息素。
      2. 最大最小蚂蚁系统（Thomas Stutzle）：①只对最优的蚂蚁进行信息素更新；②将信息素值域范围限制在一个区间内；③。。。
    5. 特点：
      1. 并行算法：每只蚂蚁在问题空间的多点同时开始独立的解搜索，不仅增加了算法的可靠性，也使得算法具有较强的全局搜索能力。
      2. 是一种自组织的算法，命令来自系统的内部。
      3. 具有较强的鲁棒性，蚁群算法的求解结果不依赖于初始路线的选择，参数较少，设置简单，易于应用到组合优化问题的求解
      4. 是一种正反馈算法。

  • 粒子群算法

    1. 内容：每个优化问题的潜在解都是搜索空间中的一个粒子，每个粒子都有一个由被优化的函数决定的适应度值，每个粒子还有一个速度，然后粒子就追随当前的最优粒子在解空间中搜索。
    2. 流程：
      1. 随机初始化粒子群，包括位置和速度；
      2. 计算适应度值；（适应度函数其实就是目标求解函数）
      3. 根据两个极值更新自身的速度和位置；
      4. 边界条件处理。
    3. 特点：
      1. 通过粒子间的合作于竞争产生的群体智能知道优化搜索，是一种高效的并行搜索算法；
      2. 粒子群算法和遗传算法都是随机初始化种群，使用适应值来评价个体，但是粒子群根据自己的速度来搜索，没有遗传的变异和交叉；
      3. 除了能找到问题的最优解之外，还能得到若干个次优解，因为每个粒子在算法结束时仍保持其个体极值。
      4. 特有的记忆使其可以动态的跟踪当前搜索情况并调整其搜索策略。
